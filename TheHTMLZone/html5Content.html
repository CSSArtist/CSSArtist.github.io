Transparent Content Model

Description

Some elements are described as Transparent; they have “transparent” in the description of their Content Model. The Content Model of a Transparent element is derived from the Content Model of its parent element: the elements required in the part of the Content Model that is “transparent” are the same elements as required in the part of the Content Model of the parent of the Transparent element in which the Transparent element finds itself. 

For instance, an ins element inside a ruby element cannot contain an rt element, because the part of the ruby elements Content Model that allows ins elements is the part that allows Phrasing ContentPhrasing Contentsection above, and the rt element is not Phrasing Content.

Paragraphs

Concept

The term “paragraph”, is used for more than just the definition of the pelement descriptionan zone document element. The paragraph concept defined here is used to describe how to interpret documents. The pElement Specificationsan zone document element is merely one of several ways of marking up a paragraph. 

A paragraph is typically a run of Phrasing ContentPhrasing Contentsection above that forms a block of text with one or more sentences that discuss a particular topic, such as Art, but can also be used for more general thematic grouping. For instance, an address is also a paragraph, as is a part of a form, a byline, or a stanza in a poem.

Footnotes

does not have a dedicated mechanism for marking up footnotes. Here are the suggested alternatives: 

  • For short, inline annotations, the title attribute could be used. 
  • Unfortunately, using on the title attribute is currently not recommented because many browsers do not make the attribute accessible, as required by the specificationsThis is to include:
    : The Living Standard
    Developers Edition
    w3c Editors Draft    . They require a pointing device such as a mouse, to cause a tooltip to appear, which excludes keyboardonly users and touchonly users, such as anyone with a modern phone or tablet.
  • If the title attribute is used, CSS can used to draw the readers attention to the elements with the attribute. 
  • For example, the following CSS places a dashed line below elements that have a title attribute. 
  • [title] { borderbottom: thin dashed; }
  • For longer annotations, the a element should be used, pointing to an element later in the document. The convention is that the contents of the link be a number in square brackets. 
  • For side notes, longer annotations that apply to entire sections of the text rather than just specific words or sentences, the aside element should be used. 
  • For the figure or table elements, footnotes can be included in the relevant figcaption or caption element, or in surrounding prose.
